# DeukPack 연재 아티클 시리즈 (총 20편)

이 시리즈는 DeukPack이 현대 개발 환경(분산 시스템, 다국어 환경, AI 코딩 어시스턴트, 게임 개발)에서 단순한 직렬화 도구를 넘어 왜 **'유니버설 스키마 멀티 허브(Universal Schema Multi-hub)'**로 자리매김하고 있는지 개발자 커뮤니티에 알리기 위해 기획되었습니다.

## 테마 1: 스키마 중심 개발(Schema-First)과 시대적 요구
* **01편:** [AI 코딩 시대, 왜 다시 '스키마'가 중요한가?](01-why-schema-first-in-ai-era.md)
* **02편:** [Protobuf, Thrift, OpenAPI를 하나로: 스키마 멀티 허브 DeukPack의 탄생](02-deukpack-birth-story.md)
* **03편:** 마이크로서비스 간의 '잃어버린 계약' 찾기: JSON의 한계와 바이너리 IDL
* **04편:** 에이전트(Agent) 워크플로에서 DeukPack이 환각(Hallucination)을 막는 법

## 테마 2: DeukPack 핵심 기술과 퍼포먼스
* **05편:** DeukPack의 심장, DpProtocol 파헤치기 (Thrift 호환과 메모리 최적화)
* **06편:** 메모리 할당 0을 향하여: C# ArrayPool과 Zero-Copy 직렬화 기법
* **07편:** O(1) 메시지 디스패치: 수백 개의 패킷 핸들러를 우아하게 관리하는 법
* **08편:** DeukPack IDL (`.deuk`) 설계 철학: 개발자 친화적인 C++ 스타일 문법
* **09편:** Elixir와 Erlang VM의 극한 최적화: 순수 바이너리 패턴 매칭 직렬화

## 테마 3: 다중 언어와 플랫폼 통합
* **10편:** C#(Unity), C++, TypeScript를 아우르는 '단일 진실 공급원(SSOT)' 구축
* **11편:** 프론트엔드(React/TS)와 백엔드의 완벽한 타입 동기화 실전
* **12편:** Entity Framework(ORM)와 IDL의 만남: 지긋지긋한 DTO 복제 끝내기
* **13편:** OpenAPI(Swagger)와 DeukPack 연동: 바이너리 RPC와 REST의 평화로운 공존
* **14편:** 기존 Thrift 레거시 시스템을 중단 없이 DeukPack으로 마이그레이션하기

## 테마 4: 게임 개발 파이프라인의 혁신
* **15편:** 게임 메타데이터와 네트워크 패킷의 스키마를 하나로 통일하기
* **16편:** 엑셀(Excel) 지옥 탈출기: 기획 데이터를 코드로 안전하게 다루는 법
* **17편:** List, Set, Map 시트 분리와 참조 무결성 검증 자동화
* **18편:** 스키마 버전 관리와 호환성: 필드 추가/삭제에도 클라이언트가 죽지 않는 이유
* **19편:** SQLite와 DeukPack을 활용한 암호화된 게임 로컬 데이터 시스템 구축
